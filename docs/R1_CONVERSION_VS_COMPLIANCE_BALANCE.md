# R1 — Conversion vs compliance balance

## The tension

- **Compliance** images (pure white, single SKU, no text) are often **less** emotionally engaging than **lifestyle** shots.
- **Marketplaces** with strict main-image rules **force** compliance on **slot 1** only; **gallery** may allow richer context — **Amazon** pattern widely cited (via Zentail → Seller Central links).

## What competitors optimize for (evidence)

| Objective | Evidence |
|-----------|----------|
| **ROAS / “compelling” creatives** | Feedonomics — templates with **banners, logos, event overlays** for **ads** |
| **Channel guidelines** | Channable — compliance language on **marketing pages**; rules fail feeds on bad generation |
| **Operational safety** | Channable — “feed will fail to run” on image errors |
| **A/B on images** | Feedonomics — feature marketing includes testing **images** as attributes |

## Inference (labeled)

- **Most multichannel cores optimize for “not breaking feeds”** first; **conversion** is **secondary** via **optional** creative tools or **off-platform** ads.
- **“Attractiveness scoring”** of supplier JPEGs is **rarely** documented; **Helium 10–class** tools push **generative** alternatives instead.

## Practical pattern: split “commerce” vs “ads”

1. **Listing pipeline** — **compliance-first** main, **policy-validated** gallery order.
2. **Ads / social dynamic creatives** — **separate** templates (Channable / Feedonomics style) with **different** policy object.

## For strict marketplaces (e.g. ML portada)

**Reasonable approach:**  
- **Main:** maximize **compliance + legibility** (plain bg, full product, no graphics).  
- **Conversion:** push to **secondary images** or **title/price/reviews** — **not** proven as automated swap in competitor docs, but **consistent** with **slot** models.
