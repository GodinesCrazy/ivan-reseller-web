## Architecture Decisions

### Secrets vs DB
- **Env / secret manager only**
  - `SHOPIFY_CLIENT_ID`
  - `SHOPIFY_CLIENT_SECRET`
- **DB metadata only**
  - `shopifyStoreUrl` as optional safe store-domain fallback
  - `shopifyLocationId`
  - pricing / margin / operational settings
- **Not used anymore for auth**
  - `shopifyAccessToken` legacy DB field remains in schema for backwards compatibility, but the module ignores it and does not accept it in config updates.

### Shopify Auth Model
- Auth mode: `client_credentials`
- Token lifetime: short-lived (refreshed by repeating the same exchange)
- Token storage: in-memory cache only
- API surface used by the backend: GraphQL Admin API

## Remaining External Blockers

1. **Storefront visibility** — the published product is active in Shopify, but anonymous access still hits the Shopify password page instead of buyer-facing PDP content.
2. **CJ credentials** — must be configured per user for Discover search/evaluate/import to work.
3. **Profit snapshots** — generated by order processing only; empty until real orders flow.
4. **Order-ingestion proof** — controlled publish is verified, but real Shopify order ingestion/tracking sync still needs its own live pass.
